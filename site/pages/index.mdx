## One Tool, Multiple Protocols

</div>

```bash
# Hyperliquid — check price, trade, stream
heat hl price BTC
heat hl buy ETH 0.5 --price 3500 --dry-run
heat hl stream trades BTC | jq '.px'

# Polymarket — browse markets, trade
heat pm markets search "election"
heat pm buy <token_id> --price 0.55 --size 100
heat pm positions --json | jq '.[] | {market, size}'

# Machine output — pipe-aware by default
heat hl balance --json | jq '.account_value'
```

## Protocol-First Design

</div>

<div style={{ fontSize: '15px', opacity: 0.8, lineHeight: 1.7, margin: '16px 0 48px' }}>
Every protocol gets its own first-class command tree — <code>heat hl</code>, <code>heat pm</code> — with native vocabulary and no forced abstractions. Heat owns the output contract, safety model, and account layer, while each protocol speaks its own language underneath.
